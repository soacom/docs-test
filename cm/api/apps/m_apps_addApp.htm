POST /api/apps Method

Creates a new app. This method should be immediately followed by the POST /api/apps/{AppID}/versions method which creates a single app version with a generated runtime ID and a default name that is the same as the AppID.

This method creates the app, but an app is not usable without at least one version. The Community Manager user interface doesn't support an app with no version; in the user interface, when an app is added a version is added also, and when the only version is deleted the app is automatically deleted also. This operation should always be immediately followed by the POST /api/apps/{AppID}/versions operation.

Authorization Roles/Permissions: User, Business Administrator

Authorization token renewal: This operation changes information that is reflected in the authorization token; therefore, when invoking this operation, you must also renew the token.

Sample Request

The example below adds a crossword puzzle app. A picture was uploaded for the app.

Sample request URL

https://<hostname>/api/apps

Sample request headers

POST /api/apps HTTP/1.1
Host: <hostname>
Accept: text/plain
Content-Type: application/json; charset=UTF-8

Sample request body

{
  "Name":"Crosfhocal",
  "Description":"Crossword puzzle app",
  "PictureID":"1001"
}
