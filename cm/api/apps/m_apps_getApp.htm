GET apps/{AppID} Method

Returns information about an app.

Authorization Roles/Permissions: App team member, Business Administrator, or Site Admin.

This topic includes the following sections:

  • HTTP Method
  • URL
  • Sample Request
  • Request Headers
  • Request Parameters
  • Response
  • Sample Response

  • Response Body
  • Error Codes/Messages

HTTP Method

GET

Back to top

URL

https://<hostname>/api/apps/{AppID}[?IncludeSettings={boolean}]

Back to top

Sample Request

The example below requests information for the specified app.

Sample request URL #1

https://<hostname>/api/apps/app16964.acmecorp

Sample request URL #2

https://<hostname>/api/apps/app16964.acmecorp?IncludeSettings=true

Sample request headers

GET /api/apps/app16964.acmecorp HTTP/1.1
Host: <hostname>
Accept: application/json, text/javascript, */*; q=0.01

Sample request body

Not applicable.

Back to top

Request Headers

For general information on request header values, refer to HTTP Request Headers.

Header Description
Accept application/json

Back to top

Request Parameters

Parameter Parm Type Data Type Required Description
AppID Path string Required The unique ID for the specified app.
IncludeSettings Query boolean Optional Indicates whether the settings for the resource should be included in the information returned. If not specified, defaults to false.

Back to top

Response

If successful, this method returns HTTP status code 200, with information about the requested app. The information is returned as a JSON object.

Response Body

Name Type Description
Application Application Information about the app, including: AppID, Name, DefaultVersionID, IsFollowed, RatingSummary, State, Created, and Updated. The response includes settings if they were requested.
