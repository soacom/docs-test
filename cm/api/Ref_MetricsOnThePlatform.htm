Metrics on the Platform

Most operations perform actions that create, modify, read, or delete platform resources such as apps, APIs, discussions, tickets, and so forth.

However, there is also a set of operations that monitors and reports on app/API transaction activity. These operations are:

  • getMetrics (for example, GET /api/apps/versions/{AppVersionID}/metrics)
  • getTransactionRecord (for example, GET /api/apps/versions/{AppVersionID}/txlogs/{TransactionID})
  • getTxLogs (for example, GET /api/apps/versions/{AppVersionID}/txlogs)
  • exportMetrics (for example, GET /api/apps/versions/{AppVersionID}/metrics/export)

Some examples of how you can use these operations, depending on your app or API admin access rights, are:

  • Query how many times an app uses a specific API in a specified timeframe
  • Query the number of requests processed by a specific API in a specified timeframe
  • If an app is using multiple APIs, query how many times it used one API or all APIs
  • Query how much time each request took

The first three operations report on the transaction information in different ways; exportMetrics exports the information in CSV format.

Use these operations to monitor your app/API activity.
