GET /api/apis/versions/{APIVersionID}/legals

Retrieves information about legal documents for the specified API version.

Authorization Roles/Permissions: To complete this operation successfully, a user must be logged in and must be an Administrator for the API.

Request Parameters

Parameter Parm Type Data Type Required Description
APIVersionID Path string Required The unique ID for the specified API version.
Drafts Query boolean Optional Optional parameter indicating whether legal documents with a status of Draft should be included in the results returned. Defaults to false; if this parameter is not specified, drafts are not included.
Inactive Query boolean Optional Optional parameter indicating whether legal documents with a status of Inactive should be included in the results returned. Defaults to false; if not specified, inactive documents are not included.

Back to top

Response

If successful, this method returns HTTP status code 200, with information about the legal agreements. The information is sent as an RSS channel in XML or JSON format.

Response Body

The response body is in the form of an RSS channel, and includes the items listed below. The RSS version is 1.0. The title of the RSS channel is List Agreement Documents. Each item in the channel represents an agreement document, and includes the information listed below.

Name Description
title The title of the legal agreement document.
link URL for the agreement document on the server.
description User-defined description of the agreement document.
category

One or more categories. Each category is a set of two name/value pairs. The second item defines the Domain, and the first defines the actual value for the property. For example, the set of values below shows that the document type is a legal eula (end-user licensing agreement) document:

{
  "value" : "com.soa.acmecorp.legals.eula",
  "domain" : "uddi:soa.com:documenttype"
}

The domains for which values are defined are:

  • Resource type: uddi:soa.com:resourcetype
  • Document type: uddi:soa.com:documenttype
  • Document State: uddi:soa.com:state
  • Filename: uddi:soa.com:filename
  • Version: uddi:soa.com:version
  • Document path: uddi:soa.com:docpath
guid GUID for the legal agreement.
EntityReferences

One or more entity references. Each includes:

  • GUID: GUID for the API version the legal agreement relates to.
