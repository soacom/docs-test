Example: Adding an App

This example walks you through adding an app to Community Platform by using the API rather than the user interface. One use of this sequence is to build code that will load multiple APIs via the platform. This could be useful if you are just getting started on Community Platform and have many APIs.

This exercise assumes that you already have a valid login for Community Platform. You can't add an app unless you're an authorized user. Before running the procedures below, go into the user interface to set up your user profile.

Essentially, you'll need to complete three activities, in sequence:

  1. Log in using your authorized user profile you've already set up. Operation: POST /api/login.
  2. Create an app. This sets up basic parameters, but you must immediately add a version. An app isn't valid without at least one version, but you need the AppID that's returned when you add an app, so that you can use it to add an app version. Operation: POST apps.
  3. Create a version. Operation: POST /api/apps/{AppID}/versions.

Once you've completed the above steps, the app is ready for use.

Follow the detailed instructions below.

To add an app in the Community Platform using the API

  1. Log in via the API, using your own email and password in the body of the operation instead of the placeholder values given below.
    • HTTP method: POST
    • URL: https://open.soa.com/api/login
    • Accept header: application/json
    • Content-Type header: application/json
    • Request Body: {"email":"MyEmailAddress","password":"MyPassword"}
  2. Add the app. In your API call, use the values below, modifying the URL path if needed, and filling your own values for name and description in the Request Body (URL encoding not needed):
    • HTTP method: POST
    • URL: https://open.soa.com/api/apps
    • Accept header: text/plain
    • Content-Type header: application/json
    • Request Body: {"Name":"MyAppName","Description":"My App Description."}
  3. Take the AppID returned in the response; you will use it for the next operation.
  4. Add the app version. In your API call, use the following, substituting the correct AppID in the URL and also in the request body:
    • HTTP method: POST
    • URL: https://open.soa.com/api/apps/{AppID}/versions
    • Accept header: text/plain
    • Content-Type header: application/json
    • Request Body: {"Name":"xxx","Description":"xxx.","Tag":["xxx","xxx"],"AppID":"appxxx.open","InitialEnvironment":"Sandbox","Visibility":"xxx"}
  5. Repeat steps 2-4 as needed to add subsequent apps.
  6. When done, remember to log out. Use the following values:
    • URL: {xxx}
    • Accept header: text/plain
    • Content-Type header: application/json
    • Request Body: {xxx}
