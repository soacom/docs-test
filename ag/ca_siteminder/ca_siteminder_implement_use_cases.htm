Using SMSESSION Cookie with HTTP Security Policy

When a user in SiteMinder protected realm is authenticated, a cookie called SMSESSION is generated. This use case will test authentication of this SMSESSION cookie using HTTP Security Policy. To generate the SMSESSION cookie, we will first use Basic Authentication.

  1. Be sure to check the SiteMinder Agent Configuration Object in SiteMinder Administrative UI has a parameter called ‘AcceptTPCookie’ with the value ‘yes’
  2. Create a Virtual Service (VS1) then virtualize this Virtual Service again (VS2). Each Virtual Service will be hosted on separate ND
  3. VS1 will have HTTP Security Policy with Basic Authentication. The policy will also have a response cookie called ‘SMSESSION’. Enable ‘preserve transport binding headers’ to preserve this cookie header downstream.
  4. VS2 will have HTTP Security Policy with Cookie Authentication.

Configure identity system (LoginProcessManager and Identity browsing not needed)
